<Linux>
1. ls  // 파일보기
2. ls -a // 전체 파일 보기, 숨긴 파일도 나온다.
3. ls -al // 전체 파일 상세보기
4. clear // 프롬프트창 깨끗하게 만들기
5. touch // 파일 생성


<Git>
add : 작동하기 전 setting 해놓는 과정  //// git add . -> 모든 파일들을 add 적용
commit -m"adad": 작동 시작, 박제 기록에 남김 (기록 명이 adad임.)
status : 상태 확인 (add가 적용되었는지 파악 가능)
checkout 해시코드 : 해시코드에 해당하는 파일 확인 /// git checkout ~ -> 기존의 기록된 버젼에서 master 버젼으로 변환.
log : add 된 상태의 파일들을 확인 가능(변경된 모든 파일의 기록을 확인 가능)
